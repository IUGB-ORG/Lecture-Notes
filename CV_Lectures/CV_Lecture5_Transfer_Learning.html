The Problem We Are Solving

Suppose you want to build a CNN that classifies X-ray images as showing pneumonia or not. You have 500 labeled examples. Training AlexNet from scratch on 500 images would produce a badly overfit, essentially useless model — AlexNet has 60 million parameters; 500 images would cover a tiny fraction of the weight space.

But here is the key insight from Lecture 4: AlexNet, trained on ImageNet, already knows how to detect edges, textures, curves, shapes, gradients, regions of high contrast — the same low-level visual features that appear in X-rays. Only the final "what does this mean?" interpretation differs between natural photos and medical images.

The Transfer Learning Thesis

Early CNN layers learn universal visual features that transfer across tasks. Only later layers learn task-specific patterns. Therefore: borrow the universal early layers from a large pre-trained model, discard the final task-specific layers, and train only the new head on your small dataset. You get the benefit of millions of training examples without actually having them.

What Each Layer Learns to See

The strongest evidence for transfer learning comes from visualizing what each layer of a trained CNN responds to. This work was pioneered by Zeiler and Fergus in their 2013 ZFNet paper — the same year as the Dogs vs. Cats competition.

What CNN layers detect, from early to deep. The hierarchy from edges → textures → parts → objects is universal across domains. Only the final layer is task-specific.

The Hierarchy of Features

LayersWhat is learnedTransferable?
Conv1–2Oriented edges, color blobs, gradients✓ Always
Conv3–4Textures, corners, repeated patterns✓ Usually
Conv5Object parts — wheels, eyes, fins~ Sometimes
FC1–2High-level semantic combinations✗ Task-specific
OutputClass probabilities for original task✗ Replace this
💡 Why This Works Intuitively

Think about learning to draw. You first learned to make straight lines and curves (low-level). Then you learned to draw eyes and noses (mid-level). Then faces (high-level). If you now need to draw a cat, you reuse lines and curves entirely, partially reuse face-parts knowledge (ears, eyes), and only need to learn the cat-specific combinations.

🔬 The Zeiler & Fergus (2013) Insight

By deconvolving activations back to pixel space, they showed that the same edge and texture detectors appear in CNNs trained on completely different datasets. The early layers are not learning "ImageNet edges" — they are learning edges, period.

Three Transfer Learning Strategies

Which strategy to use depends on two factors: how much data you have and how similar your domain is to the pre-training domain. Click each card to see when to use it.

SMALL DATA · SIMILAR DOMAIN

Feature Extraction

Freeze all pre-trained weights. Remove the final FC layer. Extract features from the frozen backbone. Train only a new classifier head.

MEDIUM DATA · ANY DOMAIN

Fine-tuning

Unfreeze some or all of the pre-trained layers. Continue training the entire network on new data — but with a very low learning rate.

LARGE DATA · DIFFERENT DOMAIN

Train from Scratch

Ignore pre-trained weights entirely. Initialize randomly. Train fully on your own large dataset. Rare — requires millions of examples.

Feature Extraction — The Classic Approach

Take a pre-trained model (ResNet, VGG, EfficientNet). Remove its final fully-connected classification layer. Pass your images through the frozen remaining layers — this produces a fixed-length feature vector for each image (e.g., 2048-dimensional for ResNet-50). Then train a simple logistic regression or 1–2 layer MLP on these feature vectors.

When to use: You have <1,000 labeled examples. Your domain is visually similar to ImageNet (photographs of real-world objects). Training time is extremely fast — often minutes to seconds because you never backprop through the heavy backbone.

The Modern Backbone — ResNet

When choosing a pre-trained backbone, ResNet (Residual Networks, He et al. 2015) is the default starting point. It introduced the residual connection — a direct shortcut that adds a layer's input to its output — solving the vanishing gradient problem in very deep networks and enabling training of 50, 101, and 152-layer CNNs.

The Residual Block

A residual block. The skip connection allows gradients to flow directly to early layers without passing through the non-linearities.

Why ResNet is the Transfer Learning Default

💡 ResNet-50 Rule of Thumb

For most practical transfer learning tasks, start with ResNet-50 pretrained on ImageNet. It is the best-studied backbone with the most tutorials, and its 2048-dimensional feature vector is rich enough for most downstream tasks. Only switch if ResNet-50 is too slow (→ ResNet-18) or accuracy is insufficient (→ ResNet-101, EfficientNet, or ViT).

Residual Block:
output = F(x) + x
where F = Conv → BN → ReLU → Conv → BN
and x is the skip connection (input)

Gradient flows: ∂L/∂x = ∂L/∂output · (∂F/∂x + 1)
The "+1" guarantees gradient ≥ 1 at every block.

Complete PyTorch Implementation

Strategy A — Feature Extraction

import torch
import torch.nn as nn
import torchvision.models as models

# 1. Load pre-trained ResNet-50
model = models.resnet50(pretrained=True)

# 2. Freeze ALL backbone layers
for param in model.parameters():
    param.requires_grad = False

# 3. Replace final FC layer
#    ResNet-50 outputs 2048-dim features
num_features = model.fc.in_features   # 2048
model.fc = nn.Sequential(
    nn.Linear(num_features, 256),
    nn.ReLU(),
    nn.Dropout(0.3),
    nn.Linear(256, num_classes)  # e.g. 2
)
# Note: model.fc is NOT frozen — it was just created
#   so requires_grad=True by default ✓

# 4. Train ONLY the new head
optimizer = torch.optim.Adam(
    model.fc.parameters(),   # ← only head params
    lr=1e-3
)

Strategy B — Fine-tuning

# 1. Load pre-trained backbone
model = models.resnet50(pretrained=True)

# 2. Replace head
num_features = model.fc.in_features
model.fc = nn.Linear(num_features, num_classes)

# 3. Freeze early layers, unfreeze late ones
#    ResNet-50 has: layer1, layer2, layer3, layer4
for name, param in model.named_parameters():
    if 'layer4' not in name and 'fc' not in name:
        param.requires_grad = False

# 4. Use differential learning rates
optimizer = torch.optim.Adam([
    {'params': model.layer4.parameters(),
     'lr': 1e-4},    # small LR for pre-trained
    {'params': model.fc.parameters(),
     'lr': 1e-3},    # larger LR for new head
])

# 5. Training loop — identical to before
for epoch in range(20):
    for images, labels in train_loader:
        optimizer.zero_grad()
        out = model(images)
        loss = criterion(out, labels)
        loss.backward()
        optimizer.step()

Data Augmentation — Essential for Small Datasets

from torchvision import transforms

# Training transforms — augment to increase effective dataset size
train_transform = transforms.Compose([
    transforms.RandomResizedCrop(224),      # random crop + resize
    transforms.RandomHorizontalFlip(),       # 50% chance of flip
    transforms.ColorJitter(
        brightness=0.2, contrast=0.2,
        saturation=0.1),                      # random color shift
    transforms.RandomRotation(15),           # ±15° rotation
    transforms.ToTensor(),
    transforms.Normalize(                    # CRITICAL: use ImageNet
        mean=[0.485, 0.456, 0.406],         # mean and std for
        std= [0.229, 0.224, 0.225])          # pre-trained models
])

# Validation transforms — NO augmentation, just resize + normalize
val_transform = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize(
        mean=[0.485, 0.456, 0.406],
        std= [0.229, 0.224, 0.225])
])
⚠️ The ImageNet Normalization Rule

Pre-trained models expect inputs normalized with ImageNet statistics: mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]. If you skip this normalization, the feature maps in the early layers will be meaningless and transfer learning will fail. Always apply these values — even if your images look different from natural photographs.

Practical Decision Guide

Transfer learning strategy selection flowchart. Start top-left and follow the decision tree based on your dataset size and domain similarity.
Scenario Dataset Strategy Learning Rate Expected Accuracy
Medical imaging (small) 200–1,000 images Feature Extraction 1e-3 (head only) 80–92%
Product classification 5,000–50,000 Fine-tune last 2 blocks 1e-4 (backbone), 1e-3 (head) 90–96%
Satellite imagery 10,000+ Full fine-tune 1e-5 (all layers) Depends on task
Dogs vs. Cats 25,000 Fine-tune 1e-4 / 1e-3 >99%
Large proprietary dataset 1M+ From scratch or fine-tune 1e-3 Task-dependent
🔑 The General Rule

When in doubt, start with Feature Extraction. It is the fastest to implement, the least likely to overfit, and often produces surprisingly good results even when you expect fine-tuning to be necessary. Then evaluate: if the features are clearly insufficient, progressively unfreeze more layers.

Key Takeaways — Lecture 5

01

Early layers = universal. Edges, textures, and shapes are the same in X-rays, satellite images, and natural photos. They always transfer.

02

requires_grad = False freezes layers. Frozen layers are not updated by backprop. Frozen ≠ inactive — they still compute forward-pass features.

03

Normalize with ImageNet stats. Mean=[0.485,0.456,0.406], Std=[0.229,0.224,0.225]. Non-negotiable for all pre-trained torchvision models.

04

Use lower learning rates for fine-tuning. The backbone already has good weights. A large LR would destroy them. Start at 1e-4 or lower.

05

Data augmentation amplifies small datasets. Random crops, flips, and color jitter can effectively multiply your training set several times over.

06

ResNet-50 is the practical default. Well-studied, widely documented, strong performance across domains. Start here unless you have a specific reason not to.
