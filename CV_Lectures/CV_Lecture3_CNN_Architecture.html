Recap & What We Are Building Today

Over the first two lectures we established the two foundational pillars of computer vision in deep learning. In Lecture 1 we learned that an image is a tensor of numbers — grayscale (H,W), color (H,W,3). In Lecture 2 we learned that convolution slides a small filter across that tensor, computing a local dot product at each position to produce a feature map — and that multiple filters in parallel produce a stack of feature maps.

Today we assemble those pieces into a complete Convolutional Neural Network: a pipeline that takes a raw pixel tensor as input and outputs a class prediction. We will trace a single image through every layer — watching the shape transform at each stage — and understand how the whole thing is trained end-to-end with backpropagation.

The Big Idea for Today

A CNN is just a sequence of five types of layer, each with a clear job. Stack them in the right order, define a loss function, run backpropagation — and the network learns to see.

The Five-Layer Pipeline

Every classical CNN — from the 1998 LeNet to modern networks — is built from the same five building blocks, applied repeatedly before a final classifier:

The canonical CNN pipeline. Each colored block is a layer type. The shape annotation shows how the tensor dimensions evolve for a 28×28 grayscale input.
#LayerJobShape change (example)
1 Conv2D Detect local features with learned filters (28,28,1) → (26,26,32)
2 ReLU Introduce non-linearity, zero out negatives (26,26,32) → (26,26,32) (unchanged)
3 MaxPool2D Downsample, keep strongest activations (26,26,32) → (13,13,32)
4 Flatten Convert 3D tensor to 1D vector for the classifier (13,13,32) → (5408,)
5 Dense + Softmax Fully connected classifier, outputs class probabilities (5408,) → (10,)
🔑 The Pattern

Blocks 1–3 are the feature extractor — they transform raw pixels into a rich, compact representation. Blocks 4–5 are the classifier — a plain MLP applied to that representation. The CNN's power lives entirely in the feature extractor. The MLP at the end is almost trivial.

The ReLU Activation Layer

You already know ReLU from our MLP work. In a CNN it plays the same role — it is applied element-wise to every value in the feature map after each convolution, introducing the non-linearity the network needs to learn complex functions.

ReLU(x)  =  max(0, x)  =  { x  if x > 0  ·  0  if x ≤ 0 }

Why ReLU After Conv?

A convolution is a purely linear operation — it can only compute weighted sums. Stack 10 conv layers without any non-linearity and you still have one big linear transformation. ReLU breaks the linearity, allowing the network to represent non-linear boundaries between classes.

ReLU also has a convenient visual interpretation in the context of feature maps: it zeroes out negative activations, leaving only positions where the filter's pattern was found. The remaining positive values are the "evidence" for a detected feature at that spatial location.

💡 Why Not Sigmoid or Tanh Here?

In deep networks, sigmoid and tanh cause the vanishing gradient problem — gradients shrink as they pass through many layers. ReLU's gradient is either 1 (if x>0) or 0, which keeps gradients alive in deep networks. This is a major reason CNNs became practical around 2012.

ReLU applied to a feature map. Negative values (dark in the center-top) are zeroed out; positive activations are preserved.

Pooling — Spatial Downsampling

After one or more Conv+ReLU blocks, the feature maps are typically still quite large. A pooling layer reduces their spatial dimensions — making the network more efficient, reducing overfitting, and introducing a degree of translation invariance.

Max Pooling

The most common pooling operation. A 2×2 window slides over the feature map with stride 2. At each position it keeps only the maximum value in the 2×2 patch — discarding the other three.

2×2 MaxPool, stride 2:
H_out = H_in / 2  ·  W_out = W_in / 2
Number of channels unchanged.

Intuition: "Was this feature detected anywhere in this 2×2 region?" The max answers yes/no — it doesn't care exactly where. This is the source of the mild translation invariance: if an edge shifts by one pixel, the max pooling output may be identical.

📐 Average Pooling

Instead of the max, take the average of the patch. Less common in classical CNNs but used in modern architectures (e.g. Global Average Pooling, which collapses an entire feature map to a single value — used instead of Flatten in ResNet and MobileNet).

2×2 Max Pooling on a 4×4 feature map. Each colored quadrant collapses to its maximum value.

Why Not Just Use Stride in the Conv Layer?

Good question — and in fact modern networks often do exactly that. Strided convolutions (stride=2) achieve the same downsampling while still learning a transformation, rather than just taking a max. In practice: older networks (LeNet, AlexNet, VGG) use explicit pooling layers; modern networks (ResNet, EfficientNet) often replace them with strided convolutions. Both approaches are valid and worth knowing.

Shape Trace — Following the Tensor

One of the most important skills in CNN programming is tracking the shape of your tensor through every layer. A shape mismatch causes an error; knowing the expected shape at each stage lets you debug instantly.

Shape evolution for a simple 2-block CNN on MNIST (28×28 grayscale, 10 classes). Each arrow shows one layer; the tensor shape is annotated above each block.

The Formula Sheet

After Conv2D (valid, stride S=1):
H_out = H_in − F + 1
W_out = W_in − F + 1
C_out = N_filters

After Conv2D (same padding):
H_out = H_in  ·  W_out = W_in
After MaxPool2D (pool=2, stride=2):
H_out = H_in / 2
W_out = W_in / 2
C_out = C_in (unchanged)

After Flatten:
Length = H_out × W_out × C_out
⚠️ The #1 CNN Debug Tip

When your code crashes with a shape error, print the shape at every layer until you find the mismatch. In PyTorch: print(x.shape) after every operation. In Keras: model.summary() prints all shapes automatically. Never guess shapes — always verify.

Full Forward Pass — Code to Architecture

PyTorch Implementation

import torch
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):

    def __init__(self):
        super().__init__()

        # ── Feature extractor ──────────────
        # Block 1
        self.conv1 = nn.Conv2d(1, 32, kernel_size=3)  # (1,28,28)→(32,26,26)
        self.pool1 = nn.MaxPool2d(2, 2)               # (32,26,26)→(32,13,13)

        # Block 2
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3) # (32,13,13)→(64,11,11)
        self.pool2 = nn.MaxPool2d(2, 2)               # (64,11,11)→(64,5,5)

        # ── Classifier ─────────────────────
        self.fc1 = nn.Linear(64 * 5 * 5, 128)          # 1600 → 128
        self.fc2 = nn.Linear(128, 10)                  # 128  → 10 classes

    def forward(self, x):           # x: (N, 1, 28, 28)

        # Block 1: Conv → ReLU → Pool
        x = self.conv1(x)           # → (N, 32, 26, 26)
        x = F.relu(x)               # → (N, 32, 26, 26)
        x = self.pool1(x)           # → (N, 32, 13, 13)

        # Block 2: Conv → ReLU → Pool
        x = self.conv2(x)           # → (N, 64, 11, 11)
        x = F.relu(x)               # → (N, 64, 11, 11)
        x = self.pool2(x)           # → (N, 64,  5,  5)

        # Flatten + Classify
        x = x.flatten(start_dim=1)  # → (N, 1600)
        x = F.relu(self.fc1(x))     # → (N, 128)
        x = self.fc2(x)             # → (N, 10)  logits
        return x                     # apply softmax in loss fn

model = SimpleCNN()
x = torch.randn(8, 1, 28, 28)  # batch of 8
out = model(x)
print(out.shape)  # → (8, 10)

💡 Where the parameters live

Notice that 91% of the parameters live in fc1 — the first dense layer after flattening. This is why Global Average Pooling (replacing Flatten+Dense with a single average per channel) is popular in modern architectures: it eliminates this dense layer almost entirely.

🔗 Connection to Our MLPs

Recognize fc1 and fc2? That's a 2-layer MLP — exactly what we built from scratch. The CNN's feature extractor produces the input vector; the MLP classifies it. Everything we learned about MLP forward passes, loss functions, and backpropagation applies here unchanged.

Training — Loss, Backprop, and What Gets Learned

Training a CNN is identical in principle to training our MLP from scratch. The algorithm is the same:

  1. Forward pass — run a batch of images through all layers → get logits (N×10 matrix)
  2. Compute loss — compare logits to true labels using Cross-Entropy loss
  3. Backward pass — PyTorch auto-differentiates through conv, relu, pool, and dense layers
  4. Update weights — optimizer (SGD or Adam) adjusts every filter and dense weight

Training Loop

import torch.optim as optim

model = SimpleCNN()
optimizer = optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.CrossEntropyLoss()

for epoch in range(10):
    for images, labels in train_loader:
        # images: (N, 1, 28, 28)  labels: (N,)

        optimizer.zero_grad()          # clear gradients
        logits = model(images)          # forward pass
        loss = criterion(logits, labels)# compute loss
        loss.backward()                 # backprop
        optimizer.step()               # update weights

Backprop Through Convolution

You do not need to derive the gradient of a convolution manually — PyTorch's autograd handles it. But conceptually it is worth knowing:

LayerGradient passes through?
Dense (fc)Yes — same as MLP
FlattenYes — trivially reshapes gradient
MaxPoolYes — only the max position receives gradient; others get 0
ReLUYes — gradient = 1 where x>0, else 0
Conv2DYes — gradient is itself a convolution (with flipped kernel)
🎓 Key Insight

The filter weights in Conv2D layers are updated by backprop just like MLP weights. Early in training, filters are random noise. After thousands of iterations they specialize — each learning to detect a specific visual pattern that helps minimize the loss.

Gradient Flow Through the Architecture

Gradients flow backward (right → left) through every layer. Each arrow carries ∂Loss/∂weights, which the optimizer uses to update that layer's parameters.

Two More Tools You Will See Everywhere

Modern CNNs typically include two additional layer types. You do not need to implement them from scratch — but you need to recognize them and know what they do.

Dropout

During training only, Dropout randomly zeroes out a fraction p of activations in a layer (typically p = 0.25–0.5). This prevents neurons from co-adapting — they cannot rely on any specific neighbor always being present.

💡 Regularization

Think of Dropout as training an ensemble of many slightly different networks simultaneously — each mini-batch uses a different subset of neurons. At test time, all neurons are active and the weights are scaled by (1−p) to compensate. The effect: dramatically reduced overfitting.

# In PyTorch
self.drop = nn.Dropout(p=0.25)

def forward(self, x):
    ...
    x = self.drop(x)   # applied after flatten
    x = self.fc1(x)
    ...

Batch Normalization

Batch Normalization (BatchNorm) normalizes the activations of a layer across the mini-batch — zero-centering and scaling them to unit variance, then learning a rescaling parameter.

💡 Why It Helps

Without BatchNorm, activations can drift to very large or very small values as they pass through deep networks, making training slow and unstable. BatchNorm keeps activations in a healthy range throughout training, allowing much higher learning rates and faster convergence.

# Typical conv block with BN
self.conv1 = nn.Conv2d(1, 32, 3)
self.bn1   = nn.BatchNorm2d(32)

def forward(self, x):
    x = self.conv1(x)
    x = self.bn1(x)    # normalize
    x = F.relu(x)
    ...
📐 Canonical Order

The standard modern block order is:
Conv → BatchNorm → ReLU → Pool

Key Takeaways

01

A CNN = Feature Extractor + Classifier. Conv+ReLU+Pool blocks extract features; a final MLP classifies them.

02

ReLU kills negatives. Applied after every conv, it provides the non-linearity needed to learn complex boundaries.

03

MaxPool halves spatial size. Reduces computation and adds mild translation invariance.

04

Track shapes at every layer. Know the formula for each layer type. Print shapes when debugging.

05

Training is identical to MLP. Cross-Entropy loss → backprop → optimizer update. Autograd handles everything.

06

Dropout + BatchNorm. Add these to any production CNN. BN before ReLU; Dropout after Flatten.
