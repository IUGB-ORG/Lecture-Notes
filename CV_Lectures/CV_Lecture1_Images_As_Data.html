Why Should We Care?

Over the past five weeks we have built neural networks from scratch — perceptrons, multi-layer perceptrons, forward passes, backpropagation. All inputs to those networks were clean numeric vectors: a handful of real-valued features neatly packaged into a column vector x.

Now consider an image. A photograph is not a column of four numbers. A single phone photo might contain 12 million individual measurements. How do we turn raw visual data into something a neural network can digest? That question is the entire subject of this lecture series on Computer Vision.

The Big Idea for Today

Every image — no matter how rich or colorful — is just a rectangular grid of numbers. Once you truly believe that, the door to computer vision swings wide open.

What is a Pixel?

The word pixel is a portmanteau: picture + element. It is the smallest addressable unit of a digital image — a tiny square that carries exactly one color.

A screen or image sensor assigns each pixel a numeric brightness value. For the most common 8-bit encoding, that number lives in the integer range:

0  ≤  pixel  ≤  255

Why 0–255?

Modern computers store color information using 8 bits per channel. Eight bits can represent 2⁸ = 256 distinct states, so the natural integer range is 0 through 255.

ValueMeaning
0Pure black — no light
128Mid-gray — half brightness
255Pure white — maximum light
💡 For Neural Networks

We almost always normalize pixel values before feeding them to a network:
x_norm = x / 255.0
This maps [0, 255] → [0.0, 1.0] — the same kind of range our activation functions prefer.

Grayscale Images — The 2D Array

A grayscale (black-and-white) image is the simplest case. Every pixel has exactly one brightness value. Arrange those values in a rectangle and you have a 2-dimensional NumPy array — or in the language of deep learning, a rank-2 tensor.

Image shape:  (H, W)

H = height in pixels  ·  W = width in pixels

Representing as NumPy

import numpy as np

# A tiny 6×6 grayscale image
img = np.array([
  [10,  30,  80, 120, 190, 240],
  [20,  50,  90, 130, 180, 230],
  [35,  70, 105, 150, 200, 245],
  [15,  45,  95, 140, 175, 220],
  [25,  60, 110, 160, 210, 250],
  [ 5,  40,  85, 145, 195, 235]
], dtype=np.uint8)

# Shape: rows × columns
print(img.shape)   # → (6, 6)
print(img.dtype)   # → uint8

# Access pixel at row=0, col=3
print(img[0, 3])   # → 120

# Normalize for neural network input
img_norm = img / 255.0  # dtype becomes float64
print(img_norm[0, 3])  # → 0.4706...

🔑 Key Notation

The element at row i, column j of a grayscale image is written:
img[i, j]
Row index 0 is the top row. Column index 0 is the left column. This is standard for NumPy and every deep learning framework.

⚠️ uint8 vs float32

uint8 stores raw pixel values (0–255, 1 byte each). For network computations, convert to float32 and divide by 255. Failing to do this is a common beginner bug that causes training to diverge.

Color Images — The 3D Tensor

A color photograph adds a new dimension. Each pixel now requires three numbers — one for each primary color of light:

R G B each ∈ [0, 255]

The result is a rank-3 tensor — three "stacked" 2D arrays, one per channel:

Image shape:  (H, W, 3)

The "3" is the number of color channels (R, G, B)

An (H, W, 3) tensor — three stacked grids, one per channel. The final color is the combination of all three.

NumPy Representation

import numpy as np

# A tiny 4×4 color image
# Shape: (4, 4, 3)
img = np.zeros((4, 4, 3), dtype=np.uint8)

# Set top-left pixel to red
img[0, 0] = [255, 0, 0]

# Set center pixel to a custom color
img[2, 2] = [200, 100, 60]

print(img.shape)        # → (4, 4, 3)
print(img[0, 0])         # → [255, 0, 0]

# Extract just the Red channel
red_channel = img[:, :, 0]   # shape (4, 4)
grn_channel = img[:, :, 1]   # shape (4, 4)
blu_channel = img[:, :, 2]   # shape (4, 4)

Channel Decomposition

One powerful way to understand color images: "un-stack" them into three independent grayscale images, each showing how much of one primary color is present at every pixel.

🎨 Why RGB?

Human eyes have three types of color receptors (cone cells) tuned to red, green, and blue wavelengths. RGB encoding mirrors this biology. Camera sensors are built the same way. Other encodings exist (HSV, LAB) but RGB is the universal default for raw image files.

📐 Channel-First vs Channel-Last

NumPy and standard image libraries use (H, W, 3) — channel-last. PyTorch uses (3, H, W) — channel-first. TensorFlow uses (H, W, 3). Always check your framework's convention!

Why Shape is Everything

In our MLP work, we always flattened inputs into a 1D vector. With images, that approach quickly becomes problematic — and understanding why motivates everything about CNNs that comes in the next lectures.

Flattening a (4×4×3) color image produces a 48-element vector. A 224×224 color image produces 150,528 inputs — making a naive MLP enormous.

The Flattening Problem

Suppose we want to classify a 224×224 color image using a plain MLP. Flattening gives us a vector of length:

224 × 224 × 3 = 150,528

If the first hidden layer has 512 neurons, that's already 77 million weight parameters — just in the first layer. Training this on small datasets is hopeless.

🔜 Preview: CNNs to the Rescue

Convolutional Neural Networks (Lecture 2–3) exploit the spatial structure of images — pixels near each other are related — to dramatically reduce parameters while preserving what matters.

Spatial Locality

A key property of images that plain MLPs ignore:

PropertyWhat it means
LocalityAdjacent pixels are usually related (same object surface)
StationarityA vertical edge looks like a vertical edge regardless of where it appears
HierarchyEdges → shapes → parts → objects
💡 Insight

When you flatten a (28, 28) image to 784 values, pixel[0] (top-left corner) and pixel[27] (top-right corner of row 0) end up as adjacent values in the vector. But pixel[28] — which is directly below pixel[0] — is now 28 positions away. The MLP has no idea these are neighbors.

NumPy Image Cheat Sheet

Every deep learning framework — PyTorch, TensorFlow, Keras — ultimately works with tensors that behave like NumPy arrays. Here is the core vocabulary you will use constantly.

import numpy as np

# ── Creating image arrays ──────────
bw = np.zeros((28, 28), dtype=np.uint8)
rgb = np.zeros((224, 224, 3), dtype=np.uint8)

# ── Shape queries ─────────────────
bw.shape       # (28, 28)
rgb.shape      # (224, 224, 3)
bw.ndim        # 2  (rank)
rgb.ndim       # 3  (rank)
rgb.size       # 150528 (total elements)

# ── Pixel access ──────────────────
bw[10, 15]          # single grayscale pixel
rgb[10, 15]         # array([R, G, B])
rgb[10, 15, 0]      # Red channel only

# ── Slicing regions ───────────────
patch = rgb[10:20, 10:20]  # 10×10 crop
r_ch  = rgb[:, :, 0]        # entire R channel

# ── Type conversion ───────────────
img_f = rgb.astype(np.float32) / 255.0
# Values now in [0.0, 1.0]

# ── Add batch dimension ───────────
# Networks expect (batch, H, W, C)
batch = img_f[np.newaxis, ...]  # (1, 224, 224, 3)

Shapes at a Glance

ArrayShapeInterpretation
bw(H, W)Grayscale image
rgb(H, W, 3)Color image (H-last)
rgb_pt(3, H, W)Color image (PyTorch)
batch(N, H, W, 3)N images (mini-batch)

Loading Real Images

from PIL import Image
import numpy as np

img_pil = Image.open("photo.jpg")
img_np  = np.array(img_pil)
print(img_np.shape)  # e.g. (480, 640, 3)
print(img_np.dtype)  # uint8

# Convert to grayscale
gray_pil = img_pil.convert("L")
gray_np  = np.array(gray_pil)
print(gray_np.shape) # (480, 640)
🔧 Common Gotcha

If you open a .jpg file saved with 4 channels (RGBA — including transparency), shape[-1] will be 4, not 3. Always check shape and convert with .convert("RGB") before processing.

Key Takeaways

01

Images are numbers. A pixel is just an integer in [0, 255]. There is no magic — only arithmetic.

02

Grayscale = 2D array. Shape is (H, W). Access with img[row, col].

03

Color = 3D tensor. Shape is (H, W, 3) — one slice per RGB channel.

04

Normalize before training. Divide by 255 to map pixels into [0.0, 1.0].

05

Shape is your compass. Always check .shape when debugging vision pipelines.

06

Flattening loses structure. Spatial relationships matter — this is why CNNs exist.
