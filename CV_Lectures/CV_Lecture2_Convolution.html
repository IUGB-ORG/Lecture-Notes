Where We Left Off

In Lecture 1 we established that every image is just a rectangular grid of numbers — a 2D array for grayscale, a 3D tensor for color. We also identified a fundamental problem: if we flatten a 224×224 image and feed it to a plain MLP, we need tens of millions of parameters just in the first layer, and we completely destroy the spatial relationships between pixels.

The solution is Convolutional Neural Networks (CNNs). And the single mathematical idea at their core is the convolution operation. That is the entire subject of this lecture.

The Big Idea for Today

Instead of connecting every neuron to every pixel, we use a small reusable filter that slides across the image, computing a local dot product at each position. Same weights, everywhere — massively fewer parameters, and spatial structure preserved.

The Filter (Kernel)

A filter (also called a kernel) is a small 2D matrix of learnable weights — typically 3×3 or 5×5. It is the "eye" of the convolution. Think of it as a small template that asks: "does this local patch of the image look like me?"

Filter shape:  (F, F)

F = filter size  (typically 3 or 5)

Each element of the filter is a weight — a number that gets learned during training via backpropagation, exactly like the weights in our MLP. Before training, they are initialized randomly. After training, each filter has learned to respond to a specific visual pattern.

📐 Key Sizes
FilterParametersCommon use
3×39Modern CNNs (default)
5×525LeNet, older models
7×749AlexNet first layer
1×11Channel mixing (advanced)

Example: a 3×3 Vertical Edge Filter

Each cell is a learnable weight. This particular filter will fire strongly when it sees a vertical edge.
🔑 Filter vs Kernel — Terminology Note

In signal processing, kernel is the precise term. In deep learning, filter is more common. They mean the same thing. You will also see weight matrix or receptive window in some texts. This lecture uses them interchangeably.

The Convolution Operation — Step by Step

Here is the complete recipe for applying a 3×3 filter to a grayscale image:

  1. Position — Place the filter over a 3×3 region of the input image (starting at the top-left corner).
  2. Multiply — Multiply each filter weight by the corresponding pixel value beneath it (element-wise).
  3. Sum — Add up all 9 products. This single number goes into the output at the current position.
  4. Slide — Move the filter one step to the right (or down at the end of a row) and repeat.
  5. Result — After visiting every valid position, the collection of output values forms the feature map.

The Math

If the input is I and the filter is K, the output at position (i, j) is:

Output[i, j] =   ∑m=0F−1   ∑n=0F−1   I[i+m, j+n]  ×  K[m, n]
where F = filter size  ·  I = input image  ·  K = kernel/filter

The Calculation

Patch × Filter element-by-element:
10×−1 + 20×0 + 200×+1
15×−2 + 25×0 + 210×+2

💡 Intuition

Left side of patch ≈ 10–15 (dark). Right side ≈ 195–210 (bright). The filter assigns −1/−2/−1 to the left and +1/+2/+1 to the right. Dark left + bright right = large positive → vertical edge confirmed.

What Filters Detect

Before training, filters are random. After training on millions of images, each filter in a CNN's first layer typically learns to detect a simple, interpretable visual primitive. Here are the classical examples:

Vertical Edge

Fires on vertical transitions: dark-left / bright-right or vice versa

Horizontal Edge

Fires on horizontal transitions: dark-top / bright-bottom

Diagonal Edge

Fires on 45° edges — lines going from bottom-left to top-right

Blur (Avg)

All weights equal  1/9 — smooths the image, reduces noise

Sharpen

Center strongly positive, neighbors negative — amplifies high-frequency details

Identity

Center = 1, all others = 0. Output is identical to input. Useful baseline.

🧠 Deep Layers See More

First-layer filters detect edges and color gradients. Second-layer filters (applied to feature maps, not raw pixels) detect textures and simple shapes. Third-layer filters detect parts of objects. By the final layers, filters respond to high-level concepts — eyes, wheels, fur. The hierarchy is learned automatically from data.

Feature Maps — Multiple Filters in Parallel

In practice, a single convolutional layer applies many filters simultaneously — not just one. Each filter produces its own independent feature map. The complete output of the layer is the stack of all these maps.

Input:  (H, W, 1) → N filters of size (F, F) → Output:  (H', W', N)

N feature maps stacked into a new 3D tensor — just like an RGB image, but the "channels" are now learned features.

One input image → 4 different filters → 4 feature maps. Each map highlights a different visual property of the original image.

Color Images: One Filter per Channel

For a color input of shape (H, W, 3), a single filter has shape (F, F, 3) — one 2D kernel per input channel. The three channel outputs are summed to produce one scalar per position, so each filter still produces exactly one feature map.

📐 Parameter Count

A conv layer with N filters of size F×F applied to C input channels has:
N × F × F × C + N parameters
(the +N accounts for one bias per filter)

Example: 32 filters, 3×3, on a grayscale image (C=1):
32 × 9 × 1 + 32 = 320 parameters

vs. Fully Connected

Compare to a dense MLP layer connecting a 28×28 grayscale image to 32 hidden neurons:

MLP:  784 × 32 + 32 = 25,120
Conv:  32 × 9 + 32 = 320

The convolutional layer achieves the same 32-channel output with 78× fewer parameters — and it generalizes better because the same filter works everywhere in the image.

✨ Weight Sharing

The same 9 weights are reused at every spatial position. If a vertical edge detector works in the top-left corner, it works in the bottom-right corner too. This is called weight sharing and it is the key efficiency gain of convolution.

Stride & Padding

Two hyperparameters control the geometry of the convolution output: stride and padding.

Stride

Stride S controls how many pixels the filter moves at each step. Stride 1 is the default (move one pixel). Stride 2 skips every other pixel — the feature map comes out roughly half the size.

Output size formula (1D):
H_out = floor( (H_in − F) / S ) + 1

Example: 28×28 input, 3×3 filter, stride 1:
H_out = (28 − 3) / 1 + 1 = 26

Same input, stride 2:
H_out = (28 − 3) / 2 + 1 = 13
💡 Why Use Stride > 1?

To downsample the spatial dimensions — reducing computation in deeper layers. Stride 2 is often used instead of max pooling in modern architectures. Think of it as zooming out.

Padding

When the filter slides off the edge of the image, valid positions end early — the output is smaller than the input. Padding adds a border of zeros around the input to control this.

ModeDescriptionEffect
'valid'No padding (default)Output shrinks by F−1
'same'Pad to preserve sizeOutput = input size
📐 Padding Formula

For 'same' output with stride 1: add P = (F−1)/2 zeros on each side. For a 3×3 filter: P = 1. For a 5×5 filter: P = 2.

A 4×4 image padded by 1 zero on each side, ready for a 3×3 convolution that preserves the 4×4 spatial size.
Stride 1 (top) vs Stride 2 (bottom) on the same 6×6 input with a 3×3 filter. Stride 2 visits half the positions and produces a smaller feature map.

The Three Key Insights of Convolution

🔗

1. Local Connectivity

Each output value depends on only a small local patch of the input (F×F pixels). A neuron doesn't need to see the whole image — nearby pixels carry the relevant spatial context.

♻️

2. Weight Sharing

The same filter weights are applied everywhere. A detector trained to find eyes in the upper-left of the image automatically works in the lower-right. This is translation equivariance.

🏛️

3. Hierarchical Features

Stack multiple conv layers and each layer sees the output of the previous. Layer 1 sees pixels → detects edges. Layer 2 sees edge maps → detects shapes. Etc. Complexity emerges from depth.

🔗 Connection to MLP

A convolution layer is a constrained version of a fully connected layer. If you took a dense layer and forced all neurons that share the same relative position to also share their weights, you'd get a convolution. The constraint is not a limitation — it is a very powerful inductive bias that encodes what we know about images: patterns repeat spatially.

Convolution in NumPy & PyTorch

Manual Convolution (NumPy)

Writing it by hand cements the understanding:

import numpy as np

def conv2d_manual(image, kernel):
    """Single-channel 2D convolution, valid padding."""
    H, W = image.shape
    F     = kernel.shape[0]     # assume square kernel
    H_out = H - F + 1
    W_out = W - F + 1
    output = np.zeros((H_out, W_out))

    for i in range(H_out):
        for j in range(W_out):
            patch  = image[i:i+F, j:j+F]  # extract F×F region
            output[i, j] = np.sum(patch * kernel)

    return output

# Vertical edge filter
kernel = np.array([
    [-1, 0, 1],
    [-2, 0, 2],
    [-1, 0, 1]
], dtype=np.float32)

img = np.random.randint(0, 255, (28, 28))
feature_map = conv2d_manual(img, kernel)
print(feature_map.shape)  # → (26, 26)  [valid padding]

In PyTorch (What You'll Use)

import torch
import torch.nn as nn

# Define a conv layer:
# in_channels=1 (grayscale)
# out_channels=32 (number of filters)
# kernel_size=3×3, stride=1, padding=1 ('same')
conv = nn.Conv2d(
    in_channels=1,
    out_channels=32,
    kernel_size=3,
    stride=1,
    padding=1
)

# Input: batch of 8 grayscale 28×28 images
# Shape: (batch, channels, H, W)
x = torch.randn(8, 1, 28, 28)
y = conv(x)
print(y.shape)  # → (8, 32, 28, 28)
#                   ↑    ↑    ↑   ↑
#                 batch  32  H   W  (preserved by padding=1)

# Count parameters:
print(sum(p.numel() for p in conv.parameters()))
# → 320   (32×9 + 32 biases)
⚠️ PyTorch Channel Convention

PyTorch expects inputs as (N, C, H, W) — channel-first. If your NumPy image is (H, W, C), convert with: x = torch.from_numpy(img).permute(2, 0, 1)

Key Takeaways

01

A filter is a small weight matrix (3×3 typical) that slides across the image computing a local dot product at every position.

02

Output = element-wise multiply + sum. Nine multiplications and eight additions per position. Simple arithmetic.

03

Each filter produces one feature map. N filters → N feature maps stacked into shape (H', W', N).

04

Weight sharing is the key efficiency. Same 9 weights reused at every position → 78× fewer parameters than a dense layer.

05

Stride controls downsampling. Stride 1 = dense scan. Stride 2 ≈ halves spatial dimensions.

06

Padding preserves size. 'same' padding keeps H×W constant; 'valid' lets it shrink by F−1.
