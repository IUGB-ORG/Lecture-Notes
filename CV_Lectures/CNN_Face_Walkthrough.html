Chapter 1 of 6

What does "seeing" mean
for a computer?

A human looks at this image and instantly recognises a face. A CNN cannot do that — at least not at first. What it can do is arithmetic. The entire story of how a CNN goes from pixels to recognition is a story of arithmetic applied repeatedly, in layers.

The Input — What We See
A portrait — shapes, colour, light, structure.
The Input — What the CNN Sees
200 × 240 × 3 = 144,000 integers
🔑 The Core Idea

There is no "face" stored anywhere in the CNN. There are only matrices of numbers being transformed by other matrices of numbers. Recognition emerges from the structure of those transformations — learned entirely from data.

Chapter 2 of 6

The filter — a question
in matrix form

A CNN filter (also called a kernel) is a tiny 3×3 grid of weights. It encodes one question: "Does this particular pattern exist at this location?" The weights determine what pattern it looks for.

The filter below is the Sobel vertical-edge detector. Look at its weight pattern carefully:

Sobel X — Vertical Edge Filter

The green (+2) column asks: "is the right side bright?" The red (−2) column asks: "is the left side dark?" When the answer to both is yes, a strong positive output fires — a vertical edge detected.

📐 Reading the Kernel

Positive weights (green): the filter rewards bright pixels in these positions.
Negative weights (red): the filter is penalised by bright pixels here.
Zero (grey): these positions are ignored.

💡 What each kernel detects

The filter's "answer" is highest when the input patch matches the filter's weight pattern — when bright pixels align with positive weights and dark pixels align with negative weights.

One step of convolution —
the actual arithmetic

We will now zoom into the eye region of the face and watch the filter compute its output at a single position. This is the operation that happens thousands of times to produce one feature map.

What is happening

The 3×3 filter is placed over a 3×3 patch of the image. Each filter weight is multiplied by the corresponding pixel value. The 9 products are summed. That sum is the output value at this position. Step through the calculation below.

Chapter 4 of 6

Scanning the whole face —
building the feature map

The filter does not run once. It slides to every position in the image — top-left to bottom-right — repeating that 9-multiply-and-sum operation at each location. The result is a complete feature map: a new image where each pixel's brightness represents how strongly that filter fired at that location.

Chapter 5 of 6

32 filters in parallel —
a bank of detectors

In a real CNN, the first layer runs not one filter but 32 (or 64) filters simultaneously, each looking for a different pattern. Together they produce 32 feature maps — one per filter. Each map answers a different question about the same image.

📌 Each feature map = one detector's answer

Below are 8 filters applied to the same face. Notice that each map looks completely different — because each filter asks a completely different question. The bright regions in each map show where that filter's pattern was found.

What the network is accumulating: By the end of the first layer, for every position in the image, the network has 32 numbers — each describing a different local feature at that location. The image has gone from 3 channels (RGB) to 32 channels of learned descriptions.

Chapter 6 of 6

Layer by layer —
from edges to a face

Here is the complete journey. Each layer takes the previous layer's output as its input — not the original image. This means later layers detect patterns in patterns, building an increasingly abstract and structured understanding.

🎯 Why this works for faces

Layer 1 detects edges — the eye socket boundary, the chin line, the lip edge. Layer 2 detects curves and junctions — where those edges meet, forming arcs like an eyebrow or a nostril curve. Layer 3 detects parts — a detector whose receptive field spans the full eye region fires when it sees: a curved arc above two white ellipses flanking a dark circle. This is what an eye is, expressed mathematically.

Why we apply ReLU after each layer

After each convolution, every value is passed through ReLU: max(0, x). This zeroes out negative values. The result: only the positions where a filter positively matched survive. Positions where the pattern was absent (negative output) become zero — silence.

output = max( 0, Σ pixel × weight )
if the filter found the pattern → positive → kept
if the filter found nothing → negative → zeroed
Before vs. After ReLU
Left: raw edge map (with negatives) · Right: after ReLU

Final output — classification confidence

After several layers of Conv+ReLU+Pool, the feature maps are flattened into a vector and passed through a fully-connected layer with softmax. The output is a probability for each class.

✓ The complete pipeline

Raw pixels → Conv1 (edges) → ReLU → Pool → Conv2 (textures/curves) → ReLU → Pool → Conv3 (parts) → Flatten → Dense → Softmax output. Every arrow is a matrix multiplication and non-linearity. No hand-crafted features anywhere.
